# Contributing Guide

## How to Contribute

Thank you for interested in contributing to Mazuri Waste Management System! Here's how you can help.

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help each other learn
- Report issues responsibly

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit with clear messages: `git commit -m 'Add description of changes'`
6. Push to your branch: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Development Workflow

### Backend Development

```bash
cd backend
npm install
npm run dev
```

### Frontend Development

```bash
cd frontend
npm install
npm start
```

## Code Style

- Use meaningful variable and function names
- Write comments for complex logic
- Follow existing code patterns
- Keep functions small and focused
- Use async/await over promises where possible

## Testing

Before submitting a PR:
- Test your changes thoroughly
- Verify no existing functionality is broken
- Test with different user roles
- Check error handling

## Pull Request Process

1. Update README.md if adding new features
2. Update INSTALLATION.md if changing setup process
3. Update API_REFERENCE.md if modifying API endpoints
4. Provide clear PR description explaining changes
5. Link related issues if applicable

## Reporting Bugs

When reporting bugs, please include:
- Clear description of the issue
- Steps to reproduce
- Current behavior vs expected behavior
- Screenshots if applicable
- Environment details (OS, browser, etc.)

## Feature Requests

- Describe the feature and why it's needed
- Provide examples of how it would be used
- Explain the expected benefit
- Check if similar features exist

## Documentation

- Update docs when adding features
- Keep examples up-to-date
- Use clear and simple language
- Include code examples

## Performance Considerations

- Optimize database queries
- Minimize API calls
- Use efficient algorithms
- Consider memory usage
- Profile before and after changes

## Security

- Never commit sensitive data
- Use environment variables for secrets
- Validate all inputs
- Follow security best practices
- Report security issues privately

## Areas for Contribution

- Bug fixes
- Feature development
- Documentation improvements
- Test coverage
- Performance optimization
- UI/UX improvements
- DevOps and deployment

## Need Help?

- Check existing issues and discussions
- Read the documentation
- Ask in pull requests
- Contact the team
